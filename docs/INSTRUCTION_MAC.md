# FlowDictate — інструкція (macOS)

> ⚠️ **Чесно про статус:** готового Mac-застосунку поки НЕМАЄ. Windows-версія
> (`FlowDictate.exe`) на Mac не запуститься в принципі — це інша платформа.
> Ядро розпізнавання (Whisper) кросплатформне, тому переписувати логіку не
> треба, але системні частини (гаряча клавіша, вставка тексту, значок у
> меню-барі, звук) на macOS робляться іншими бібліотеками. Це окрема збірка,
> орієнтовно день роботи.
>
> Ця інструкція описує: (А) що потрібно й скільки це буде «коштувати» за
> швидкістю, і (Б) два шляхи запуску на Mac, коли візьмемось.

---

## 1. Важливе про швидкість на Mac (перевірено, не здогадка)

На Mac немає відеокарти NVIDIA, тому двигун із Windows-версії (faster-whisper)
там працює **тільки на процесорі** — прискорення через Apple Metal у цьому
двигуні відсутнє. На нових Mac (Apple Silicon M1–M4) процесор потужний і
використовує вбудований Apple Accelerate, тож буде прийнятно, але не так
миттєво, як на твоїй NVIDIA (де 6 секунд мовлення розпізнаються за ~0.6 с).

Тому для Mac є два шляхи, і вони по-різному швидкі:

| Шлях | Двигун | Швидкість на Apple Silicon | Складність |
|------|--------|-----------------------------|-----------|
| **А. Той самий двигун** | faster-whisper (CPU) | Помірна | Простіше (ядро вже є) |
| **Б. Швидший двигун** | whisper.cpp + Metal | Швидко (~реальний час на large) | Трохи більше налаштувань |

Рекомендація: почати з **шляху А** (менше змін), а якщо швидкість на
конкретному Mac не влаштує — перевести двигун на **Б** (whisper.cpp з Metal
використовує графіку Apple і на M-чипах помітно швидший).

## 2. Що потрібно на Mac (вимоги)

- macOS 12 (Monterey) або новіша.
- Бажано Apple Silicon (M1/M2/M3/M4) — на старих Intel-Mac буде повільно.
- Мікрофон.
- Homebrew (менеджер пакетів для Mac) — для встановлення залежностей.
- Дозволи, які macOS попросить дати вручну (див. крок 5): **Мікрофон**,
  **Спеціальні можливості** (Accessibility) та **Моніторинг вводу** (Input
  Monitoring) — без них глобальна клавіша й вставка не працюватимуть.

## 3. Підготовка середовища (спільна для обох шляхів)

Відкрий термінал (Terminal) і виконай:

```bash
# 1. Homebrew (якщо ще немає)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. Python та системні бібліотеки для звуку/аудіо
brew install python@3.11 portaudio ffmpeg

# 3. Робоча папка та віртуальне середовище
mkdir -p ~/FlowDictate && cd ~/FlowDictate
python3.11 -m venv .venv
source .venv/bin/activate
```

## 4. Встановлення двигуна

### Шлях А — faster-whisper (простіше, CPU)

```bash
pip install faster-whisper sounddevice pynput pyobjc rumps
```

Що це за пакети (заміна Windows-специфічних):
- `faster-whisper` — той самий двигун розпізнавання, що на Windows.
- `sounddevice` — запис із мікрофона (кросплатформний, як на Windows).
- `pynput` — глобальна гаряча клавіша (на Mac замість Windows-бібліотеки
  `keyboard`).
- `pyobjc` — доступ до системного буфера обміну й вставки на macOS.
- `rumps` — значок у меню-барі (на Mac замість `pystray`).

### Шлях Б — whisper.cpp + Metal (швидше)

```bash
brew install whisper-cpp          # двигун з підтримкою Apple Metal
pip install sounddevice pynput pyobjc rumps
```

Тут розпізнавання викликає окремий двигун `whisper-cpp`, який використовує
графіку Apple. Обгортка (клавіша, запис, вставка, меню-бар) — та сама, лише
крок «розпізнати» іде через whisper.cpp замість faster-whisper.

## 5. Дозволи macOS (обов'язково — інакше не працюватиме)

macOS з міркувань безпеки блокує глобальні клавіші й вставку, доки ти явно не
дозволиш. Після першого запуску відкрий:

**Системні параметри → Конфіденційність і безпека**, і додай застосунок
(термінал або зібраний FlowDictate) у три списки:
1. **Мікрофон** — щоб чути голос.
2. **Спеціальні можливості (Accessibility)** — щоб друкувати текст у чужі вікна.
3. **Моніторинг вводу (Input Monitoring)** — щоб ловити гарячу клавішу глобально.

Після додавання — перезапусти застосунок.

## 6. Як користуватись (коли зібрано)

Так само, як на Windows: тримаєш гарячу клавішу → говориш → відпускаєш → текст
друкується. На Mac зручна клавіша для утримання — наприклад **right option
(⌥)** або комбінація `cmd+shift`. Значок стану буде в меню-барі вгорі
(а не в треї, як на Windows).

## 7. Чого НЕ буде «з коробки» на першому кроці

- Готового `.app` з подвійним кліком — спершу запуск із терміналу
  (`python flowdictate_mac.py`). Пакування в `.app`/`.dmg` — окремий крок
  (аналог того, що на Windows зробив PyInstaller; на Mac це `py2app`).
- Підпису Apple (нотаризації) — без неї macOS при першому запуску попередить
  «застосунок від невідомого розробника»; відкривається через праву кнопку →
  «Відкрити».

---

### Підсумок для Mac одним рядком

Ядро готове й кросплатформне; на Mac треба замінити ~4 системні модулі
(клавіша, вставка, меню-бар, — і за бажанням швидший двигун whisper.cpp),
видати 3 дозволи й зібрати `.app`. Скажи, коли брати Mac у роботу, — і на
якому саме Mac (модель чипа), щоб одразу обрати шлях А чи Б.
